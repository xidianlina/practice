## 1.为什么用Redis？
>(1).性能:在碰到需要执行耗时特别久，且结果不频繁变动的SQL，就特别适合将运行结果放入缓存。这样，后面的请求就去缓存中读取，使得请求能够迅速响应。                
 (2).并发:在大并发的情况下，所有的请求直接访问数据库，数据库会出现连接异常。这时就需要使用redis做一个缓冲操作，让请求先访问到redis，而不是直接访问数据库。                   
 (3).redis提供了相当丰富的string、hash、list、set、zset等数据结构。               
 (4).redis具备可以做分布式锁的功能。                 
 (5).redis支持数据的备份。
